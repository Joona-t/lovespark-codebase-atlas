Silent failure modes

🤖 LLM-drafted from deterministic signal · override via audit-meta.yml

- If a provenance edge gets written before its target node is committed, lookups quietly return null and downstream agents treat absence as "no such fact" rather than "race condition" — corruption that type-checks fine. - Compaction that merges two records with subtly different unit conventions (e.g. tokens-vs-chars, UTC-vs-local timestamps) will produce a single "canonical" record with plausible numbers and no warning. - Typed-absence sentinels (None, Unknown, Withheld) collapsing to the same falsy value somewhere in the pipeline — an agent that should distinguish "we asked and they refused" from "we never asked" silently can't. - 313k LOC with zero declared deps means the dependency surface lives in imports and shell calls; a renamed CLI (`claude` → `claude-cli`) or a transitive stdlib behavior change degrades one agent's output to empty strings that serialize fine. - Last commit 2026-03-28, 47 days stale: any external schema this depends on (arXiv API, GitHub API, an LLM CLI's JSON output shape) may have drifted, and the failure mode is usually "parses, returns partial, no exception." - Honestly, I can't tell from a name + LOC + commit count whether the compaction layer has idempotency tests; if it doesn't, re-running compaction on already-compacted state is the classic silent data-loss path here.

The Takeaway

🤖 LLM-drafted from deterministic signal · override via audit-meta.yml

The signal here is thin — 313k lines, 106 commits, no declared deps, and a tagline about "typed absence and provenance" is all I have to go on. So take this with appropriate doubt. The one thing worth stealing, if the description is accurate: treating absence as a typed value rather than null/None/missing-key. Most projects discover halfway through that "we don't know yet" and "we know it's not there" and "we forgot to fetch this" are three different states that got collapsed into one falsy value, and then spend the rest of their life paying for it. Naively you reach for <code>Optional</code> and move on. The less naive move is to name the reasons absence happens in your domain and let the type system carry them. You should be nervous about doing this everywhere — it gets verbose fast — but doing it at the one or two boundaries where provenance actually matters (cache layers, agent memory, anything that compacts state) tends to pay for itself. I can't tell from file counts alone whether primordial actually pulls this off, or whether the compaction story is load-bearing or aspirational. Worth a closer read before borrowing.
